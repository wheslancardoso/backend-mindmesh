feat(chat): Implementa chat com sessão e refatora arquitetura de dados

Esta é uma grande refatoração que introduz sessões de chat persistentes e melhora significativamente a arquitetura de dados e serviços.

### Principais Mudanças:

- **Chat com Sessão:**
  - Adiciona modelos `ChatSession` e `ChatMessage` e seus respectivos repositórios para persistir o histórico de conversas.
  - O endpoint `/api/chat` agora é stateful, aceitando um `sessionId` opcional e retornando o ID da sessão para continuidade.
  - A `ChatService` foi reescrita para orquestrar a criação e atualização de sessões e mensagens.

- **Arquitetura de Dados e Repositórios:**
  - O schema do banco (`init-db/01-init-schema.sql`) foi expandido para incluir as tabelas de chat e o modelo `Document` foi refinado com `file_hash` para deduplicação e `status` de processamento.
  - Implementa `DocumentChunkRepositoryCustom` para busca vetorial com projeção para DTO (`ChunkSearchResult`), evitando problemas de mapeamento do tipo `vector` com Hibernate.

- **Melhorias nos Serviços:**
  - `DocumentIngestionService` agora gerencia um fluxo de status (`pending` -> `completed`/`failed`) e previne uploads duplicados com base no hash do arquivo.
  - `EmbeddingService` reintroduz o "modo mock": se `OPENAI_API_KEY` não for definida, o serviço gera embeddings simulados sem custo, facilitando o desenvolvimento local.

- **API e DTOs:**
  - Renomeia `question` para `message` em todos os DTOs de chat para maior clareza.
  - O endpoint de upload (`/api/documents/upload`) agora retorna um `DocumentUploadResponseDto` com informações detalhadas.
  - Remove o endpoint `/reprocess` para simplificar a API.
